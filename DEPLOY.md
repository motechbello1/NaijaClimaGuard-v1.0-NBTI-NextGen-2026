# NaijaClimaGuard — Deployment Guide

## Your folder must contain these files before deploying:

```
naija-clima-guard/
  app.py
  database.py
  payments.py
  data_pipeline.py
  model_trainer.py
  flood_model.pkl        ← generated by model_trainer.py
  training_data.csv      ← generated by data_pipeline.py
  shap_bar.png           ← generated by model_trainer.py
  shap_beeswarm.png      ← generated by model_trainer.py
  confusion_matrix.png   ← generated by model_trainer.py
  requirements.txt       ← this deployment package
  .gitignore             ← this deployment package
```

---

## Step 1 — Install Git (if you don't have it)

Download from: https://git-scm.com/download/win
Install with all default options.

---

## Step 2 — Create a GitHub account

Go to https://github.com and sign up (free).

---

## Step 3 — Create a new GitHub repository

1. Click the green **New** button on GitHub
2. Name it: `naija-clima-guard`
3. Set to **Public**
4. Do NOT initialise with README (leave all boxes unchecked)
5. Click **Create repository**

---

## Step 4 — Push your code to GitHub

Open PowerShell inside your `naija-clima-guard` folder and run these commands one by one:

```powershell
git init
git add .
git commit -m "Initial commit - NaijaClimaGuard v1.0"
git branch -M main
git remote add origin https://github.com/YOUR_GITHUB_USERNAME/naija-clima-guard.git
git push -u origin main
```

Replace `YOUR_GITHUB_USERNAME` with your actual GitHub username.

When prompted for username and password:
- Username: your GitHub email
- Password: use a Personal Access Token (not your GitHub password)
  - Go to GitHub → Settings → Developer Settings → Personal Access Tokens → Tokens (classic) → Generate new token
  - Check "repo" scope → Generate → copy the token → paste as password

---

## Step 5 — Deploy on Streamlit Cloud

1. Go to https://share.streamlit.io
2. Sign in with your GitHub account
3. Click **New app**
4. Select:
   - Repository: `naija-clima-guard`
   - Branch: `main`
   - Main file path: `app.py`
5. Click **Deploy**

Streamlit Cloud will install all packages from `requirements.txt` automatically.
Deployment takes 3-5 minutes.

---

## Step 6 — Add your Paystack keys (secret — never in GitHub)

In Streamlit Cloud:
1. Open your deployed app → click the **⋮ menu** (top right) → **Settings**
2. Click **Secrets**
3. Paste this exactly:

```toml
[paystack]
secret_key = "sk_live_YOUR_REAL_SECRET_KEY"
public_key  = "pk_live_YOUR_REAL_PUBLIC_KEY"
```

4. Click **Save** — the app will restart with live payments enabled

Get your keys from: https://dashboard.paystack.com/#/settings/developer

---

## Step 7 — Create the .streamlit folder locally (for running locally too)

```powershell
mkdir .streamlit
copy secrets.toml.example .streamlit\secrets.toml
```

Then edit `.streamlit/secrets.toml` with your real keys.

---

## Step 8 — Set up the config.toml

```powershell
mkdir .streamlit
copy config.toml .streamlit\config.toml
```

## Updating the app after deployment

Every time you make a change locally:

```powershell
git add .
git commit -m "describe what you changed"
git push
```

Streamlit Cloud auto-redeploys within 60 seconds.

---

## Important notes

- The SQLite database (`naijaclimagard.db`) is created fresh on each deployment.
  Demo users are seeded automatically on startup.
- `flood_model.pkl`, `training_data.csv`, and the PNG files MUST be in the repo.
  They are needed at runtime. They are not in .gitignore.
- Never commit `.streamlit/secrets.toml` — it is in `.gitignore` for this reason.
